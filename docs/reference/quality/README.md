# Quality Reference

This folder holds the lightweight quality references used during OSS contribution triage on this branch.

## Index

1. [PR Impact Checklist](./pr-impact-checklist.md) for changed behavior, proof files, required checks, and patch-and-merge safety
2. [Design System Reference](./design-system.md) for shared UI ownership, reusable component scope, and design-system review questions

## Suggested order

1. Start with [PR Impact Checklist](./pr-impact-checklist.md) to capture the smallest authoritative proof for the change.
2. Use [Design System Reference](./design-system.md) when the PR touches shared UI, shell chrome, navigation, theme rules, or reusable components.
3. Pair these docs with [OSS Contribution Triage](../operations/oss-contribution-triage.md) and [PR Decision Rubric](../operations/pr-decision-rubric.md) when making the final maintainer call.
